# qa-check SKILL

## 입력
- `output/{시리즈명}/slide_XX.png` (최종 PNG)
- `series/{시리즈명}/copy.md`, `research.md`, `design-brief.md`
- 참조: shared/brand-identity.md

## 출력
- `series/{시리즈명}/qa-report.md`
- 이슈 목록 (심각도·위치·내용·수정 지시)

---

## 실행 흐름

### Step 1 — 기술 검증
- [ ] 캔버스 크기: 캐러셀 1080×1350 / 스토리 1080×1920
- [ ] 텍스트가 캔버스 밖으로 나가지 않았는가?
- [ ] 최소 폰트 28px 이상인가?
- [ ] 외부 URL 참조 없이 standalone HTML인가?
- [ ] CSS 하드코딩 값(색상 hex, px 직접 기재)이 없는가? (var(--) 미사용)

### Step 2 — 콘텐츠 검증

**팩트 대조**
- 슬라이드의 모든 수치가 research.md에 존재하는가?
- 출처 없는 최상급 표현이 없는가?

**카피-이미지 일치**
- 각 슬라이드에서 카피와 이미지가 같은 방향을 가리키는가?

**브랜드 톤**
- brand-identity.md 금지 표현이 사용되지 않았는가?
- 강조: 슬라이드당 2개 이내, strong 1개 이내인가?
- 이모지: 슬라이드당 2개 이내인가?

### Step 3 — 나노젠 이미지 검증 (HIGH 규칙)
- [ ] [HIGH] 인물 등장: 젠아 레퍼런스가 사용되었는가? (임의 생성 금지)
- [ ] [HIGH] 협찬 제품: 실제 제품과 색상·형태·로고가 일치하는가?
- [ ] @gena_feed 무드(Korean aesthetic, muted tones)가 유지되었는가?
- [ ] Garment Swap: 모델 정체성·배경이 보존되었는가?

### Step 4 — 디자인 품질 검증 (Playwright MCP 활용)

가능한 경우 Playwright MCP로 HTML 슬라이드를 브라우저에서 열어 시각적으로 검증한다:

**4a. 개별 슬라이드 검증**
- 텍스트 오버레이 가독성: 배경 위 텍스트가 읽히는가?
- 텍스트-이미지 간섭: 인물 얼굴/제품 위에 텍스트가 겹치지 않는가?
- 여백 균형: 콘텐츠가 한쪽으로 치우치지 않았는가?

**4b. 시리즈 전체 리듬 검증**
- 10장 연속 나열했을 때 시각적 단조로움이 없는가?
- 텍스트 중심 ↔ 이미지 중심이 교대하는가?
- 배경색/톤이 monotone으로 느껴지지 않는가?

**4c. 다양성 검증**
- 레이아웃 패턴이 최소 3가지 사용되었는가?
- 같은 레이아웃 2연속이 없는가?
- 인물 슬라이드의 앵글이 다양한가? (같은 포즈/구도 반복 체크)

### Step 5 — 이슈 리포트 작성
```
## QA Report — {시리즈명}

### HIGH (즉시 수정 — developer에게 전달)
- [slide_XX] [이슈 내용] → 수정 지시

### MID (수정 필요)
- [slide_XX] [이슈 내용] → 수정 지시

### LOW (기록 후 다음 회차 반영)
- [이슈 내용]

### DESIGN (디자인 품질 — MID 이상 시 수정)
- [slide_XX] [이슈 내용] → 수정 지시

HIGH + MID + DESIGN(MID이상) = 0건이면 → 오케스트레이터에게 "QA 통과" 보고
```

---

## 자기검증 체크리스트
- [ ] Write·Edit·Bash 도구를 사용하지 않았는가?
- [ ] 모든 슬라이드를 빠짐없이 검토했는가?
- [ ] HIGH 이슈를 MID로 낮게 분류하지 않았는가?
- [ ] 수정 지시를 구체적으로 작성했는가?
- [ ] 디자인 품질 검증(Step 4)을 수행했는가?
- [ ] 시리즈 전체 리듬과 다양성을 확인했는가?
- [ ] HIGH + MID + DESIGN(MID이상) = 0건 확인 후 통과 보고했는가?
