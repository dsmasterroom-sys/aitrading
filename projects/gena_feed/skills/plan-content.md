# plan-content SKILL

## 입력
- `series/{시리즈명}/research.md` (확정 주제 + 팩트 풀)
- 게시물 유형 및 슬라이드 수
- 참조: shared/brand-identity.md, shared/content-patterns.md

## 출력
- `series/{시리즈명}/plan.md`
- 내러티브 스레드 + 슬라이드별 8요소 명세 + 콘텐츠 밸류 맵

## 실행 흐름

### Step 0.5 — 3막 구조 아크 설계 (내러티브 스레드보다 먼저)
슬라이드 기획 전에 전체 스토리 아크를 먼저 잡는다.

**Act 1: 훅 & 공감 (슬라이드 1-2)**
- 관찰 가능한 문제 상황 (추상적 X → 구체적 장면)
- 감정 동일시 (자조/분노/당혹감 중 택 1)
- 갈등 제시 — 무엇이 문제를 반복시키는가?

**Act 2: 전개 & 발견 (슬라이드 3-7)**
- 반전 팩트 — 예상을 뒤엎는 1개 (Level 3 정보 여기 배치)
- 시도→실패→깨달음 흐름 — 일반 해결책 시도 → 부족함 발견 → 진짜 해법
- 정보를 스토리에 녹이기 — 데이터를 캐릭터 상황/대사로 전환

**Act 3: 해소 & 액션 (슬라이드 8-10)**
- 비포/애프터 대비 — 변화 시각화
- 실행 허들 낮추기 — "지금 당장 할 수 있는 1가지"
- 다음편 갈망(Cliffhanger) — 구체적 손실/호기심 유발 (❌ "다음편 — OO 가이드" / ✅ "OO를 모르면 XX한다. 다음편에서...")

**스토리 유형 레퍼런스** (택 1):
- **여정형**: 문제→시행착오→깨달음→솔루션
- **비밀폭로형**: 의외의 진실→증거→숨겨진 이유→대응법
- **타임라인형**: 과거→현재→미래 예측

### Step 1 — 내러티브 스레드 설계
기획의 핵심. 캐러셀을 "10장짜리 미니 콘텐츠"가 아니라 **하나의 이야기**로 만든다.

**1a. 한 줄 내러티브 정의** (필수)
```
"[타겟의 현재 상태] → [전환 계기] → [새로운 관점/행동]"
예: "벚꽃 시즌에 항상 늦게 준비하는 MZ → 올해는 날짜를 알면 다르다 → D-day 역산 + OOTD 미리 준비"
```

**1b. 저장 유인 포인트 3개 이상 배치** (필수)
팔로워가 "나중에 다시 봐야지"라고 느끼는 정보를 최소 3곳에 배치한다:
- **수치/날짜**: 구체적 숫자(개화일, 통계, 가격)
- **체크리스트/비교표**: 한눈에 보는 정리
- **실행 가능 팁**: "이렇게 하면 된다"는 구체적 행동 가이드
- **트렌드 키워드**: 검색하고 싶어지는 용어 소개

### Step 2 — 감정 곡선 + 콘텐츠 밸류 맵 + 정보 가치 레벨
감정 곡선에 **각 슬라이드가 제공하는 구체적 가치**를 함께 설계한다.

**정보 가치 레벨** (각 정보 슬라이드에 레벨 태깅 필수):
- **Level 1 (기본)**: 검색하면 나옴 → 최소화
- **Level 2 (해석)**: 데이터 + 의미 부여 → 기본
- **Level 3 (독점/통찰)**: 교차분석·역발상·숨은맥락 → **필수 1개 이상**

```
slide-01 (표지/훅)  : 온도 ●●●●○  가치: [스크롤 멈춤 — 반전 문구 or 충격 수치]
slide-02 (공감)     : 온도 ●○○○○  가치: [공감 — "나도 이랬어" 상황 묘사]
slide-03 (전환)     : 온도 ●●●○○  가치: [인사이트 — 모르면 손해인 팩트 1개]
slide-04~06 (정보)  : 온도 ●●○○○  가치: [핵심 정보 — 저장할 만한 구체적 데이터]
slide-07~08 (증거)  : 온도 ●●●●○  가치: [시각적 증거 — 비포/애프터 or 실물 비교]
slide-09 (실천)     : 온도 ●●●○○  가치: [액션 플랜 — "오늘 당장" 할 수 있는 것]
slide-10 (CTA)      : 온도 ●●●●●  가치: [다음편 예고 — 궁금증 유발]
```

### Step 3 — 슬라이드별 기획서 작성 (8요소)
각 슬라이드에 아래 8가지를 명시한다 (카피 문장은 쓰지 않는다):

| 항목 | 설명 |
|------|------|
| 역할 | 공감 / 전환 / 정보 / 증거 / 실천 / CTA |
| 핵심 메시지 | 이 슬라이드에서 전달할 **한 가지 핵심** (방향성, 문장 아님) |
| 저장 유인 | 이 슬라이드를 저장하게 만드는 요소 (없으면 "없음" 명시) |
| 정보 밀도 | HIGH(수치·비교표) / MID(인사이트·팁) / LOW(무드·감성) |
| 레이아웃 패턴 | content-patterns.md에서 선택 |
| 감정 온도 | ●○ 5단계 |
| 이미지 방향 | designer 참고용 — 앵글·장소·무드 1-2줄 (Step 5 앵글 프리셋 참조) |
| research.md 근거 | 해당 슬라이드의 팩트 출처 (정보/증거 슬라이드 필수) |

> **정보 밀도 규칙**: HIGH 슬라이드가 최소 3개 이상이어야 한다. LOW만 연속 3개 이상 배치 금지.

### Step 4 — 레이아웃 제약 확인
- 같은 레이아웃 2연속 → 교체
- 같은 온도 3연속 → 조정
- 이미지 방향에서 같은 앵글 2연속 → 교체

### Step 5 — 스타일링 슬라이드: 시즌 트렌드 반영
패션·코디 슬라이드 기획 시 반드시 현재 시즌 트렌드를 반영:
- WebSearch로 현재 시즌 S/S 또는 F/W 트렌드 컬러·스타일 확인
- 룩 베이스는 뉴트럴(블랙·차콜·그레이), 포인트 컬러 1개씩 시즌 트렌드 컬러 적용
- 스타일 키워드(고프코어·스트릿·미니멀 등)도 현재 시즌 트렌드에서 선택
- 각 룩마다 다른 포인트 컬러를 사용해 슬라이드 간 시각적 변화 확보
- **룩별 차별화**: 각 룩이 다른 TPO(Time-Place-Occasion)를 커버해야 한다

### Step 6 — 시리즈 연결 고리
단발성 게시물이 아니라 시리즈 콘텐츠로 설계:
- 마지막 슬라이드에 **다음편 예고 키워드** 1개 명시
- 이전 시리즈가 있다면 연결 가능한 **콜백 포인트** 명시
- 시리즈 해시태그 제안 (brand-identity.md 참조)

## 자기검증 체크리스트
- [ ] **3막 구조 아크**(Act1 공감→Act2 전개→Act3 해소)가 설계되었는가?
- [ ] **내러티브 스레드**가 한 줄로 정의되었는가?
- [ ] **저장 유인 포인트**가 최소 3곳에 배치되었는가?
- [ ] 감정 곡선이 공감→전환→정보→증거→실천→CTA 흐름을 따르는가?
- [ ] **정보 밀도 HIGH** 슬라이드가 최소 3개 이상인가?
- [ ] **Level 3 (독점/통찰) 정보**가 최소 1개 포함되었는가?
- [ ] Act 2에 **예상을 뒤엎는 반전 팩트**가 있는가?
- [ ] Act 3 CTA에 **다음편 갈망(Cliffhanger)**이 구체적인가?
- [ ] 같은 레이아웃 2연속 / 같은 온도 3연속 / 같은 앵글 2연속 없는가?
- [ ] 카피(실제 문장)를 쓰지 않고 방향성만 명시했는가?
- [ ] 모든 정보/증거 슬라이드의 근거가 research.md에 존재하는가?
- [ ] 각 슬라이드에 8요소가 모두 기재되었는가?
- [ ] 스타일링 슬라이드가 각각 다른 TPO를 커버하는가?
- [ ] 다음편 예고 키워드가 마지막 슬라이드에 있는가?
