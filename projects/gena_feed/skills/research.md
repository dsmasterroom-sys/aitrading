# researcher SKILL

## 입력
- 게시물 유형: 캐러셀 / 릴스 / 스토리
- 키워드 힌트: (없으면 자율 판단)
- 참조: shared/brand-identity.md

## 출력
- `series/{시리즈명}/research.md` 파일 (오케스트레이터가 대신 저장)
- 추천 주제 TOP 3 + 선정 근거
- 리서치 팩트 풀 (수치·출처·인사이트)

---

## 실행 흐름

### Step 1 — 트렌드 스캔

#### 1a. 주제 힌트가 있을 때
→ 바로 Step 2로 이동. 아래 도구로 팩트 보강만 수행.

#### 1b. 주제 힌트가 없을 때 — 3소스 교차 검증
아래 순서로 트렌드를 수집하고, **2개 이상 소스에서 겹치는 키워드**를 후보로 선정한다.

| 순서 | 도구 | 용도 | 호출 예시 |
|------|------|------|----------|
| ① | **Naver DataLab** (naver-search MCP → `search_datalab_trending`) | 연령·성별별 검색어 트렌드 (20-30대 여성 필터) | `startDate/endDate` 최근 7일, `timeUnit: date`, `ages: ["2","3"]`, `gender: "f"` |
| ② | **Naver 블로그/뉴스 검색** (naver-search MCP → `search_blog`, `search_news`) | 실시간 화제성·콘텐츠 방향 파악 | 키워드별 `display: 10`, `sort: sim` |
| ③ | **Naver 쇼핑인사이트** (naver-search MCP → `search_shopping_insight`) | 제품 카테고리 트렌드·검색량 추이 | 패션/뷰티 카테고리 키워드 |
| ④ | **WebSearch** | Google Trends·글로벌 트렌드 교차 검증 | `"[키워드] trend 2026"` 등 |

#### 1c. 결과 정리
- 수집된 키워드를 **빈도순** 정렬
- @gena_feed 타깃(MZ 20-35세 여성)과 교차점 확인
- 시즌·이슈·화제성 맥락 메모

### Step 2 — 주제 후보 생성
팔로워 Profit 4가지 기준으로 각 후보 평가:
- 정보: "몰랐던 걸 알게 됐다" → 저장율 직결
- 재미: "저장·공유하고 싶다" → 확산 직결
- 공감: "나만 이런 게 아니구나" → 팔로우 직결
- 호기심 해결: "왜 그런지 알고 싶었어" → 댓글 직결
→ 2개 미만 충족 시 후보 제외

### Step 3 — 협찬 제품 정보 보완 (마켓링크 있을 때)
- shared/products/{제품폴더}/product-info.md 확인
- 마켓링크가 있으면 WebFetch로 해당 페이지 스크래핑
- 수집 대상: 상세 스펙, 소재 상세, 리뷰 핵심 키워드, 배송 정보
- 수집 결과를 오케스트레이터에게 전달 → 사용자 확인 후 product-info.md 보완

### Step 4 — 벤치마크 계정 분석 (주간 자동 / 수동 트리거)

**언제 실행:**
- 주간 자동 실행 (매주 월요일 09:00, Cron)
- 신규 시리즈 기획 전 (마보스님 요청 시)
- 성과 부진 시 (저장률 < 1.5%)

**분석 대상:**
- @1ldk_shop
- @oncuration
- @eun_noo
- @theopen_product
- @nobordersshop

**분석 항목:**
1. **슬라이드 구조 패턴** (최신 5개 게시물)
   - 훅 → 정보 → CTA 배분 비율
   - 정보 밀도 HIGH 슬라이드 위치 (몇 번?)
   - 평균 슬라이드 수

2. **훅 카피 공식**
   - 숫자 + 반전 vs 질문형 vs 충격 비율
   - 구체적 예시 3개 추출

3. **정보 가치 레벨 분석**
   - Level 1 (기본) / Level 2 (해석) / Level 3 (독점) 비율
   - 차별화 전략 패턴

4. **비주얼 훅 유형**
   - 슬라이드 01 주요 유형 (풀블리드/클로즈업/타이포 등)
   - 색온도 분포 (Golden Hour / Overcast / Bright Sunny)

5. **호기심 루프 패턴**
   - 슬라이드 간 연결 전략 (질문 남기기 / 반전 예고 / 숫자 떡밥 비율)

6. **인게이지먼트 전략**
   - 댓글 유도 CTA 패턴
   - 스토리 연계 비율
   - 평균 댓글/저장 추정

**출력 형식:**
- `docs/benchmarks/weekly-YYYY-MM-DD.md`
- 종합 인사이트 + @gena_feed 적용 액션 (HIGH/MID/LOW 우선순위)

**자동 검증:**
다음 시리즈 research.md 제출 시, Jarvis가 최신 벤치마크 리포트의 "즉시 적용" 항목이 반영되었는지 자동 체크한다.

---

### Step 5 — 협찬 연계 가능성 검토
- 해당 주제가 협찬 콘텐츠로 발전 가능한가?
- shared/products/ 현재 등록 제품과 자연 연결되는가?

### Step 6 — 팩트 수집 (선정 주제 기준)
- 수치·통계 수집 (출처 반드시 명기)
- 핵심 인사이트 3-5개 정리
- 팩트 오류 위험 요소 표시

**차별화 정보 발굴 5가지 방법** (Level 3 정보를 만드는 법):
1. **데이터 교차분석**: 2개+ 데이터를 연결해 새 인사이트 도출 (A+B→C)
2. **역발상**: "당연한 것" 5개 나열 → "진짜 그럴까?" 질문 → 뒤집을 수 있는 1개가 훅
3. **숨은 맥락**: 팩트 뒤의 "왜?" 3번 반복 → 트렌드 리포트·뉴스 교차 확인
4. **비교 프레임**: 과거vs현재, 잘못된 방법vs올바른 방법 — 가치를 시각적으로 명확화
5. **실전 테스트**: 직접 실험 or 기존 UGC 리서치 → 숫자로 결과 제시

### Step 7 — 결과 정리
추천 주제 TOP 3 형식:
```
## 주제 1: [주제명]
- Profit 분석: [충족하는 2개 이상 명시]
- 예상 저장율 근거: [이유]
- 협찬 연계 가능성: [있음/낮음 + 이유]

## 리서치 팩트 풀
- [수치] 출처: [URL 또는 매체명]
- 인사이트 1~5
```

- [ ] 모든 수치에 출처가 명기되었는가?
- [ ] 기획·카피 내용이 전혀 포함되지 않았는가?
