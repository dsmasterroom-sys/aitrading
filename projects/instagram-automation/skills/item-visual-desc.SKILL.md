# item-visual-desc.SKILL.md

**용도**: 아이템 시각 특징을 프롬프트 언어로 변환  
**사용 에이전트**: item-researcher  
**버전**: 1.0

---

## 🎯 목적

items.json의 visual_desc 필드 작성 규칙.
프롬프트에 삽입될 시각적 특징을 측정 가능한 언어로 기술.

---

## 📋 작성 규칙

### 필수 포함 (3~5가지)

1. **핏**: "오버사이즈 / 슬림핏 / 루즈핏 / 크롭"
2. **디테일**: "앞단추 7개 / 지퍼 / 버튼 / 포켓"
3. **길이**: "크롭 / 허리라인 / 힙라인 / 무릎 위"
4. **소재감**: "코튼 / 데님 / 니트 / 실크"
5. **특징**: "카라 넓이 / 소매 길이 / 벨트 유무"

### 예시

**상의 (셔츠)**:
```
"오버사이즈 핏, 앞단추 7개, 카라 넓이 4cm, 소매 7부, 화이트 코튼"
```

**하의 (데님)**:
```
"와이드 핏, 연청 워싱, 허리 하이웨이스트, 길이 발목, 5포켓"
```

**아우터 (재킷)**:
```
"크롭 길이, 오버사이즈 숄더, 지퍼 클로저, 블랙 레더, 벨트 포함"
```

---

## 🚫 금지 표현

- ❌ "예쁜", "멋진", "트렌디" (주관적)
- ❌ "고급스러운", "세련된" (추상적)
- ❌ "인기 많은", "베스트" (관계없음)
