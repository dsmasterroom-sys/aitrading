  }

## 🔧 실행 흐름

### 1단계: 필요 아이템 파악

plan.md에서 콘텐츠 주제 읽기:
```
주제: "봄 데님 룩 3가지 스타일링"
→ 필요 아이템: 상의, 데님, 아우터, 가방
```

**카테고리 매핑**:
- OOTD → 상의, 하의, 아우터, 신발
- 데이트룩 → 원피스 or 상하의, 가방, 액세서리
- 오피스룩 → 재킷, 블라우스, 팬츠, 신발

### 2단계: 데이터 소스 크롤링

**우선순위**:
1. 무신사 (패션)
2. 지그재그 (트렌디)
3. 쿠팡 (가격대)
4. 네이버 쇼핑 (종합)

**무신사 크롤링**:
```python
# WebFetch 사용
url = "https://www.musinsa.com/ranking/best"
params = {
    "age": "20대",
    "gender": "여성",
    "category": "상의"
}

# 또는 검색
search_url = "https://www.musinsa.com/search/musinsa/goods"
query = plan.md에서 추출한 키워드  # 예: "오버사이즈 셔츠"
```

**수집 정보**:
- 제품명
- 브랜드
- 가격
- 제품 이미지 URL (최소 800px 이상)
- 구매 링크

### 3단계: 가격대별 필터링

**타겟별 가격대**:
- 20대 초반 / 대학생: 3만원대
- 20대 후반 / 직장인: 5~10만원대
- 30대 / 프리미엄: 10만원 이상

**콘텐츠당 권장**:
- 저가 2개 + 중가 1개
- 또는 중가 2개 + 고가 1개

### 4단계: 제품 이미지 URL 검증

**필수 조건**:
- ✅ 정면 이미지
- ✅ 배경 깔끔 (단색 or 스튜디오)
- ✅ 해상도 800px 이상
- ✅ HTTPS 프로토콜
- ✅ 실제 다운로드 가능 (404 아님)

**불합격 이미지**:
- ❌ 모델 착용 이미지 (인물 포함)
- ❌ 배경 복잡 (거리, 카페 등)
- ❌ 각도가 있는 이미지 (측면, 뒷면만)
- ❌ 저해상도 (500px 미만)

**이미지 다운로드 테스트**:
```python
import requests
response = requests.get(ref_image_url)
if response.status_code == 200 and len(response.content) > 50000:
    # 50KB 이상이면 유효
    pass
```

### 5단계: visual_desc 작성

**시각적 특징을 프롬프트 언어로 변환**:

**핵심 요소 (최소 3가지)**:
1. 핏: "오버사이즈 / 슬림핏 / 루즈핏 / 크롭"
2. 디테일: "앞단추 7개 / 지퍼 / 버튼 / 주름"
3. 길이: "크롭 / 허리라인 / 힙라인 / 무릎 위"
4. 소재감: "코튼 / 데님 / 니트 / 실크"
5. 특징: "카라 넓이 4cm / 소매 7부 / 벨트 포함"

**예시**:
```json
{
  "name": "오버사이즈 코튼 셔츠",
  "visual_desc": "오버사이즈 핏, 앞단추 7개, 카라 넓이 4cm, 소매 7부, 화이트 코튼"
}
```

**작성 규칙**:
- 측정 가능한 특징 우선
- 모호한 표현 금지 ("예쁜", "멋진" 등)
- 3~5가지 특징 (너무 많으면 비효율)

### 6단계: 제휴 링크 생성 (선택)

**쿠팡파트너스**:
```python
# API 또는 수동 변환
original_url = "https://www.coupang.com/..."
affiliate_link = "https://link.coupang.com/a/..."
```

**네이버 쇼핑**:
- 스마트스토어 상품은 파트너 신청 필요
- 링크 변환 후 items.json에 추가

## ✅ 자기검증 체크리스트

items.json 작성 완료 후 **모두 체크**:

### [ ] 1. 실제 구매 가능
- 품절 아님
- 구매 링크 유효 (200 응답)

### [ ] 2. 가격 정확
- 최신 정보 (24시간 이내)
- 할인가 있으면 할인가 기록

### [ ] 3. 제품 이미지 URL 유효
- HTTP 200 응답
- 이미지 크기 50KB 이상
- 다운로드 테스트 완료

### [ ] 4. visual_desc 구체적
- 최소 3가지 특징 포함
- 측정 가능한 표현 사용

### [ ] 5. 브랜드/제품명 정확
- 오타 없음
- 공식 명칭 사용

### [ ] 6. 가격대 적합
- 타겟 오디언스에 맞는 가격대
- 콘텐츠당 저/중/고가 적절 분포

### [ ] 7. 카테고리 일치
- type 필드 정확 ("상의", "하의", "아우터" 등)
- plan.md의 필요 아이템과 일치

### [ ] 8. 제휴 링크 (선택)
- 유효한 파트너스 링크
- 원본 URL과 동일 상품 확인

## 📊 데이터 소스별 특징

### 무신사
- **장점**: 패션 전문, 트렌디, 제품 이미지 고품질
- **크롤링**: 랭킹, 브랜드별, 카테고리별
- **가격대**: 3~20만원 (중가)

### 지그재그
- **장점**: 20대 여성 타겟, 빠른 트렌드 반영
- **크롤링**: 인기순, 찜순
- **가격대**: 2~10만원 (저~중가)

### 쿠팡
- **장점**: 저가 라인업 풍부, 빠른 배송
- **크롤링**: 로켓배송 상품, 평점순
- **가격대**: 1~5만원 (저가)

### 네이버 쇼핑
- **장점**: 종합 비교, 최저가 검색
- **크롤링**: 가격비교, 리뷰순
- **가격대**: 전 범위
