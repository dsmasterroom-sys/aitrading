## Session Greeting Response Delay Issue

### Problem
- 마보스님 reported that greeting responses were delayed
- Same greeting prompt sent 3 times (02:09, 02:09, 02:10) before I responded on the 3rd attempt
- User experienced no response to first two messages

### Diagnosis
- Ran `openclaw status` - gateway normal (PID 1306, 12ms response)
- Checked logs - no obvious errors at 02:09-02:10 timeframe
- Current session (`agent:main:main`) shows normal operation

### Root Cause (추가 확인됨)
- **잘못된 memory 로드**: 폐기 예정 프로젝트(`gena_feed_claude_project`)의 메모리를 가져옴
- **컨텍스트 혼란**: 신규 프로젝트와 무관한 내용 로드로 응답 지연 및 혼란 발생

### Proposed Solutions
1. **Immediate**: Lightweight greeting first, then background memory load
2. **Memory hygiene**: 폐기 프로젝트 메모리는 즉시 백업/아카이브 처리
3. **Best practice**: Avoid heavy tool calls during greeting - do quick reply first

### Notes
- AGENTS.md already specifies reading memory files at session start
- This may conflict with greeting instruction to keep it simple (1-3 sentences, no internal steps)
- Need to balance context loading vs. fast first response

**실제 작동**:
- developer가 patterns 완전 무시
- 매번 새로운 CSS를 하드코딩으로 작성

**비유**:
- 설계: 레고 블록(patterns) 조립 → 다음번 재사용 가능
- 실제: 매번 점토로 새로 만듦 → 다음번 또 만들어야 함
