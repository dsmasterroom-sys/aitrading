# research-trend SKILL

## 실행 흐름

### Step 1. 수집 소스 4곳 병렬 탐색

| 소스 | 수집 항목 |
|------|----------|
| 인스타 해시태그 | 급상승 태그, 저장수 높은 게시물 주제 |
| 네이버/구글 자동완성 | 뷰티·패션 연관검색어 |
| 경쟁 계정 분석 | 최근 30일 고성과 게시물 주제 |
| 시즌·이슈 캘린더 | 다음 2주 내 시즌 키워드 |

### Step 2. 주제 후보 12개 생성

GPT 프롬프트:
```
시즌: [INPUT]  타겟: 20-30대 여성, 패션/뷰티 관심
아래 4가지 Profit 유형별로 주제 3개씩 추천 (총 12개)
① 정보 (몰랐던 꿀팁)
② 재미 (공감/유머)
③ 공감 (나만 그런 게 아니었어)
④ 호기심 해결
각 주제: 인스타 게시물 제목 형태 + 저장/공유 이유 한 줄
```

### Step 3. 각 주제에 메타데이터 태그

```
- profit_type: 정보 | 재미 | 공감 | 호기심
- format_hint: 릴스 | 캐러셀 | 둘다
- monetize:    slant | 파트너스 | 광고 | 없음
- trend_score: 상 | 중 | 하
- season_fit:  높음 | 보통
```

### Step 4. 오케스트레이터에 결과 전달

researcher는 Write 권한 없음 → 결과를 오케스트레이터에게 전달
오케스트레이터가 `weekly/research.md`에 저장

## 자기검증 체크리스트

- [ ] 12개 중 profit_type 4가지 균등 분포 (각 3개)
- [ ] slant 자연 연결 가능한 주제 최소 2개 포함
- [ ] 시즌/트렌드 연결 주제 최소 3개 포함
- [ ] 제목이 바로 훅(Hook)이 되는 형태인지 확인
- [ ] 콘텐츠 기획·카피 작성 금지 (리서치 데이터만 제공)
